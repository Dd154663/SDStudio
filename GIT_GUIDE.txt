[2] 일상적으로 쓰는 명령어 (사용자용)
================================================================================

아래 명령어들은 모두 프로젝트 폴더에서 실행합니다.
  cd C:\Users\ii011\Downloads\SDStudio-3.5.0.b

--- 현재 상태 확인 ---

  git status

  → 어떤 파일이 수정되었는지 보여줌.
  → "nothing to commit, working tree clean"이면 저장할 것 없음.

--- 변경 내용 자세히 보기 ---

  git diff

  → 어떤 줄이 바뀌었는지 빨간색(삭제)/초록색(추가)으로 보여줌.
  → 너무 길면 q 키로 빠져나옴.

--- 수정 사항 저장 (커밋) ---

  git add -A
  git commit -m "여기에 무엇을 바꿨는지 한줄 설명"

  예시:
  git add -A
  git commit -m "이미지 생성 해상도 옵션 추가"

  → add -A : 모든 변경된 파일을 "저장 준비" 상태로 올림
  → commit : 실제로 세이브 슬롯에 저장함
  → -m 뒤의 메시지 : 나중에 "이때 뭘 했더라?" 할 때 볼 수 있는 메모

--- 저장 이력 보기 ---

  git log --oneline

  → 커밋 목록을 한줄씩 보여줌.
  → 예시 출력:
     abc1234 이미지 생성 해상도 옵션 추가
     157a3a7 SDStudio 3.5.0 개인 수정판 - 액세스 토큰 로그인 기능 추가

--- 수정한 것 되돌리기 (아직 커밋 안 한 경우) ---

  특정 파일만 되돌리기:
    git checkout -- src/renderer/componenets/ConfigScreen.tsx

  전체를 마지막 커밋으로 되돌리기:
    git checkout -- .

  주의: 되돌리면 수정한 내용이 사라집니다! 신중하게.

--- 특정 커밋으로 되돌리기 (커밋한 후) ---

  git log --oneline     ← 먼저 돌아가고 싶은 커밋 해시(abc1234 같은 것) 확인
  git revert abc1234    ← 해당 커밋의 변경사항을 "취소하는 새 커밋"을 만듦

  → revert는 이력을 보존하면서 안전하게 되돌리는 방법.

[4] 브랜치 활용법 (고급 - 큰 기능 추가 시)
================================================================================

브랜치 = "세이브 슬롯을 분기하는 것". 실험적인 수정을 할 때 유용합니다.

--- 새 브랜치 만들기 ---

  git checkout -b 새기능이름

  예시: git checkout -b add-batch-download
  → 현재 상태를 복사한 새 브랜치가 생기고, 그쪽에서 작업 시작.

--- 브랜치 목록 보기 ---

  git branch

  → * 표시가 현재 브랜치.

--- 원래 브랜치로 돌아가기 ---

  git checkout master

  → 새 브랜치에서의 수정은 master에 영향 없음.

--- 새 브랜치의 작업을 master에 합치기 ---

  git checkout master
  git merge 새기능이름

  → 새 기능이 잘 동작하면 master에 합침.

--- 새 브랜치 삭제 (합친 후) ---

  git branch -d 새기능이름

--- 실전 예시 ---

  큰 기능을 추가할 때:
    git checkout -b add-something    ← 분기
    (코드 수정)
    git add -A && git commit -m "설명"
    (테스트 성공)
    git checkout master              ← 메인으로 복귀
    git merge add-something          ← 합치기
    git branch -d add-something      ← 분기 삭제

  실험이 실패했을 때:
    git checkout master              ← 메인으로 복귀 (수정사항 무시됨)
    git branch -D add-something      ← 분기 강제 삭제

[5] 자주 하는 실수와 해결법
================================================================================

문제: "Author identity unknown" 에러
해결: git config user.name "ii011"
      git config user.email "ii011@local"
      (--global 없이 실행하면 이 저장소에만 적용됨)

문제: git add할 때 "error: nul" 관련 에러
원인: Windows 예약 파일명. .gitignore에 nul이 포함되어 있어야 함.
해결: .gitignore에 "nul" 줄이 있는지 확인

문제: git add할 때 "adding embedded git repository" 경고
원인: externals/capacitor-background-mode 안에 .git 폴더가 있음
해결: rm -rf externals/capacitor-background-mode/.git

문제: "LF will be replaced by CRLF" 경고
해결: 무시해도 됨. Windows 줄바꿈 방식 차이. 동작에 영향 없음.

문제: 실수로 node_modules를 커밋했다
해결: git rm -r --cached node_modules
      git commit -m "node_modules 제거"
      (.gitignore에 node_modules가 있으면 이후에는 무시됨)
